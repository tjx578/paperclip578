# Precision Alpha Engine

A board-governed AI trading operating system that produces validated alpha packets, maintains disciplined risk governance, and separates analysis, verdict, and live execution authority. Validated through four stages: backtest, optimization, paper trading, controlled live rollout.

## Architecture Layers

1. **Control Plane** — Paperclip (company, org tree, tasks, approvals, audit trail, run history, board governance)
2. **Analytical Backbone** — Unified Core Pipeline L1–L15 (context, confluence, probability, readiness, verdict, closure, memory)
3. **Capability Plane** — Tuyul Kartel Wolf Arsenal API v4.0 (market data, ML prediction, signals, backtesting, risk analytics, journal, alerts, monitoring)
4. **Governance Plane** — ULTRA (verdict), PROTOKOL (audit), WOLF (risk gate), QRCE (integrity), MEMVAULT (archive), FEEDGUARD (data gate)
5. **Execution Boundary** — External account governor, broker/MT5 boundary, controlled deployment layer

## Operating Doctrine

1. Top-down MTA mandatory: MN → W1 → D1 → H4 → H1
2. H4 body close mandatory before H1 trigger is valid
3. D1 decision gate is anchor utama setup
4. SMC is precision enhancer, not structure replacement
5. No-trade is a valid professional output
6. Stop-loss is mandatory
7. Analysis does not own final account-state sizing authority
8. Live execution must pass through external boundary

## Deliverables

- **Daily Execution Brief** — macro posture, focus/blocked pairs, session windows, ready/watch/no-trade classification
- **Board Decision Packet** — executive summary, alpha thesis, risk posture, evidence chain, invalidation logic, board ask
- **Validation Readiness Report** — backtest/optimization/paper-trading status, risk acceptance, go/no-go recommendation
- **Weekly Strategic Review** — improvements, failures, lesson extraction, system weakness, next-week priorities

## Validation Roadmap

| Stage | Goal |
| ----- | ---- |
| 1. Historical Backtesting | Validate strategy logic, measure return/drawdown/win-rate/profit-factor |
| 2. Parameter Optimization | Find robust parameters, avoid overfitting, test stability across regimes |
| 3. Paper Trading | Test live behavior, measure slippage/latency/alerting/workflow discipline |
| 4. Controlled Live Rollout | Small capital, strict monitoring, kill-switch active, instant rollback |

Each stage requires WOLF and PROTOKOL sign-off before advancing.

## Definition of Success

- Company produces board-ready packets consistently
- Setup selection quality improves over time
- No-trade discipline increases
- Backtest and paper trading show stability
- Risk discipline stays in the correct layer
- Live rollout (if performed) maintains full governance

## Definition of Failure

- Raw analysis treated as final decision
- Risk authority leaks to analysis layer
- Issue tree not used for milestone governance
- Scoring systems conflict without normalization
- Board receives noise instead of decisions
- Persona overrides governance
